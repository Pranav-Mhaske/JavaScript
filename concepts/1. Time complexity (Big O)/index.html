Time Complexity in JavaScript

Big O Notation

Big O notation is a mathematical notation used to describe the upper bound of an algorithm's time complexity. It provides a way to analyze and compare the efficiency of algorithms by expressing how their runtime grows as the input size increases.

Common Time Complexities

  • O(1) - Constant Time
  • O(log n) - Logarithmic Time
  • O(n) - Linear Time
  • O(n log n) - Linearithmic Time
  • O(n^2) - Quadratic Time
  • O(2^n) - Exponential Time
  • O(n!) - Factorial Time

Examples

- O(1): Accessing an element in an array by index.

- O(log n): Binary search in a sorted array.

- O(n): Iterating through an array to find a specific element.

- O(n log n): Sorting an array using merge sort or quicksort.

- O(n^2): Nested loops, such as bubble sort.

- O(2^n): Recursive algorithms that solve problems by dividing them into smaller subproblems, such as the Fibonacci sequence.

- O(n!): Algorithms that generate all permutations of a set, such as the traveling salesman problem.
