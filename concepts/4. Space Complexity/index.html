Space Complexity in JavaScript

Big O Notation

Big O notation is a mathematical notation used to describe the upper bound of an algorithm's space complexity. It provides a way to analyze and compare the efficiency of algorithms by expressing how their memory usage grows as the input size increases.

Common Space Complexities

  • O(1) - Constant Space
  • O(log n) - Logarithmic Space
  • O(n) - Linear Space
  • O(n log n) - Linearithmic Space
  • O(n^2) - Quadratic Space
  • O(2^n) - Exponential Space
  • O(n!) - Factorial Space

Examples

- O(1): when only a fixed amount of space is used, regardless of the input size, such as using a few variables to store intermediate results.

- O(log n): when the space used grows logarithmically with the input size, such as in recursive algorithms that divide the problem into smaller subproblems, like binary search.

- O(n): when the space used grows linearly with the input size, such as when creating a new array of the same size as the input array.

- O(n log n): when the space used grows as a combination of linear and logarithmic factors, such as in certain sorting algorithms that require additional space for merging, like merge sort.

- O(n^2): when the space used grows quadratically with the input size, such as when creating a 2D array to store pairwise combinations of elements from the input array.

- O(2^n): when the space used grows exponentially with the input size, such as in recursive algorithms that divide them into smaller subproblems, like the Fibonacci sequence.

- O(n!): when the space used grows factorially with the input size, such as in algorithms that generate all permutations of a set, like the traveling salesman problem.
