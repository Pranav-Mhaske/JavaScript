Time Complexity in JavaScript

Omega Notation

Omega notation is a mathematical notation used to describe the lower bound of an algorithm's time complexity. It provides a way to analyze and compare the efficiency of algorithms by expressing how their runtime grows as the input size increases.

Common Time Complexities

  • Ω(1) - Constant Time
  • Ω(log n) - Logarithmic Time
  • Ω(n) - Linear Time
  • Ω(n log n) - Linearithmic Time
  • Ω(n^2) - Quadratic Time
  • Ω(2^n) - Exponential Time
  • Ω(n!) - Factorial Time

Examples

- Ω(1): Accessing an element in an array by index.

- Ω(log n): Binary search in a sorted array.

- Ω(n): Iterating through an array to find a specific element.

- Ω(n log n): Sorting an array using merge sort or quicksort.

- Ω(n^2): Nested loops, such as bubble sort.

- Ω(2^n): Recursive algorithms that solve problems by dividing them into smaller subproblems, such as the Fibonacci sequence.

- Ω(n!): Algorithms that generate all permutations of a set, such as the traveling salesman problem.
